# 画面設計書: おでかけ記録×育成アプリ(仮称)

- バージョン: 1.2(ドラフト)
- 作成日: 2026-07-17 / 更新日: 2026-07-18
- 前提: REQUIREMENT.md v1.4 / DATABASE.md v1.2 に準拠
- 対象: iOS / Android(Webはv1対象外)

---

## 1. 画面設計の原則

1. **チェックインは3タップで完了**: タップ1=FAB → タップ2=POI選択 → タップ3=シャッター(=登録完了)。タグ・メモは結果画面での任意追記(原則1)
2. **相棒が主役のマップ**: 現在地マーカー自体が相棒キャラ。追加UIで地図を遮らない
3. **演出はスキップ可能**: 抽選演出はフルスクリーンだがワンタップでスキップできる(記録が主・ゲームは従)
4. **タブは4つ**: 地図(ホーム) / 図鑑 / アルバム / マイページ。チェックインFABは地図タブのみに表示

## 3. オンボーディングフロー(S-01〜S-04)

```
S-01 スプラッシュ
  ├─ セッションあり → S-10 地図ホーム
  └─ セッションなし → S-02
S-02 サインアップ/ログイン
  └─ 新規登録成功 → S-03(既存ログインは S-10 へ直行)
S-03 権限リクエスト(位置情報 → カメラ → 通知 の順に説明画面付きで)
  └─ S-04
S-04 ホームエリア設定
  └─ 完了 → S-10 地図ホーム
```

### S-03 権限リクエスト
- 各権限の**利用目的説明画面**をOSダイアログの前に挟む(ストア審査基準準拠。REQUIREMENT 8章)
  - 位置情報: 「近くのスポットを見つけて記録するために使います。使用中のみ・追跡はしません」
  - カメラ: 「その場で撮った写真だけを記録できるアプリです」
  - 通知: 「お知らせは週1回まで。毎日は送りません」
- 位置情報・カメラを拒否した場合もオンボーディングは完了できる(S-10で機能無効状態を表示。6章)
- 通知拒否はゲームに影響しない(そのまま進行)

### S-04 ホームエリア設定

```
┌──────────────────┐
│ ホームエリアを設定 │
│ ┌──────────────┐ │
│ │   [地図]      │ │
│ │      ⌂       │ │ ← 現在地から自動設定
│ │   ドラッグで   │ │    ピンをドラッグで微調整可
│ │   微調整可     │ │
│ └──────────────┘ │
│ 遠出ボーナスの基準地│
│ になります。変更は  │
│ 90日に1回までです。 │
│ ╭──────────────╮ │
│ │  ここに設定    │ │
│ ╰──────────────╯ │
└──────────────────┘
```

## 4. チェックインフロー(S-10 → S-12 → S-13 → S-14 → S-15)

### S-10 地図ホーム

```
┌──────────────────┐
│ ⚠オフライン(同期待ち2件)│ ← オフライン時のみ表示
│                  │
│   [Google Maps]  │
│  📍     📍       │ ← 訪問済みスポットピン
│        🐾        │ ← 現在地マーカー=相棒キャラ
│  📍              │    (歩行方向を向く。タップで一言)
│            ╭───╮ │
│            │📷 │ │ ← チェックインFAB(タップ1)
│            ╰───╯ │
├──────────────────┤
│ 地図  図鑑  ｱﾙﾊﾞﾑ  ﾏｲ │
└──────────────────┘
```

- ピン: チェックイン由来とアルバム由来でデザインを分ける(アルバムは色違い/フチなし)
- ピンタップ → S-11 スポット詳細
- 相棒マーカータップ → 一言セリフ吹き出し(ゲーム的ご褒美、機能はなし)
- **FABの状態**:
  - 通常: カメラアイコン
  - 1日10件到達: 無効化+タップで「今日の記録上限に達しました。また明日!」トースト
  - 位置情報/カメラ権限なし: 無効化+タップで設定導線モーダル(6章)

### S-12 POI候補選択(タップ2)

```
┌──────────────────┐
│ いまどこにいる?    │
│ ┌──────────────┐ │
│ │☕ ブルーボトル  │ │ ← 半径100mのPOI候補
│ │   コーヒー  20m │ │    (距離順、系統アイコン付き)
│ ├──────────────┤ │
│ │🏛 ○○美術館 45m │ │
│ ├──────────────┤ │
│ │🍜 ラーメン△ 80m │ │
│ └──────────────┘ │
│ ──────────────── │
│ ✎ リストにない場所を │ ← 手動入力フォールバック
│   自分で入力       │    (POI 0件/API障害時は自動でこちらを前面に)
└──────────────────┘
```

- POI選択 → 即カメラ起動(S-13)
- 手動入力: スポット名テキスト入力 → カメラ起動。「キャラ抽選は場所の確認後になります」を小さく表示(DATABASE.md resolve_manual_checkin)
- 候補0件/API障害時: リスト部分に「近くのスポットを取得できませんでした」+手動入力を主導線に切り替え

### S-13 カメラ撮影(タップ3)

- 標準カメラUI(シャッター・イン/アウト切替のみ。**ライブラリ選択ボタンは置かない**)
- シャッター → **その時点で登録完了**(バックグラウンドでアップロード+RPC呼び出し)
- オンライン時: S-14 抽選演出へ
- オフライン時: S-15 相当の簡易確認「記録しました。キャラは電波が戻ったらお楽しみ!」→ S-10へ

### S-14 抽選演出

- フルスクリーン演出(新規キャラ: 出現アニメーション / 被り: おやつ変換の短い演出)
- **画面のどこをタップしてもスキップ**して S-15 へ
- 初都道府県バッジ等の実績が同時発生した場合は演出後にバッジ獲得カードを重ねる

### S-15 チェックイン結果

```
┌──────────────────┐
│   ✓ 記録しました   │
│ ┌──────────────┐ │
│ │  [撮影写真]    │ │
│ └──────────────┘ │
│ ☕ ブルーボトルコーヒー│
│ 🐣 カフェチュン(★2) │ ← 獲得キャラ or おやつ
│ +40 XP (遠出×2)   │ ← 獲得XPと適用ボーナス
│ ──────────────── │
│ #タグを追加  ✎メモ  │ ← 任意追記(タップで入力欄展開)
│ ╭──────────────╮ │
│ │    とじる      │ │ → S-10 地図ホームへ
│ ╰──────────────╯ │
└──────────────────┘
```

- タグ・メモは後から S-11 スポット詳細の記録編集でも追記できる(DATABASE.md update_checkin_note)

```
┌──────────────────┐
│ 図鑑    52種中 12種 │
│ [全部|カフェ|グルメ|…]│ ← 系統フィルタ(横スクロール)

│ │★1││★2││★3││★4│ │    未入手はシルエット+★のみ
│ └──┘└──┘└──┘└──┘│
│ (カフェ・スイーツ)  │
│ ┌──┐┌──┐┌──┐┌──┐│
│ │🍙││🐟││⬛││⬛│ │
```

- セルタップ: 入手済み → S-21 / 未入手 → 「◯◯系統のスポットで出会えるかも」ヒントのみ表示

### S-21 キャラ詳細

```
┌──────────────────┐
│ ← カフェチュン ★2   │
│    [キャラ画像]     │
│  進化段階 ●●○ (2/3)│
│ ──────────────── │
│ XP    180 / 300   │ ← 次の進化までのバー
│ おやつ  22 / 35    │ ← 換算値の充足バー
│ ╭──────╮╭───────╮│
│ │おやつを││ 進化する ││ ← 進化ボタンは条件充足時のみ活性
│ │あげる  ││(あと13) ││
│ ╰──────╯╰───────╯│
│ ╭──────────────╮ │
│ │ 相棒にする      │ │
│ ╰──────────────╯ │
│ ✦ このキャラと出会った│
│   場所: ○○(記録へ) │ ← 入手チェックインへのリンク
└──────────────────┘
```

- 「おやつをあげる」: ランク別在庫(×個数)を選んで与えるシート。与えた分が snack_value_fed に加算(DATABASE.md feed_snacks)
- 「進化する」: 条件充足時のみ活性。タップでフルスクリーン進化演出(スキップ可)
- 最終進化(3段階目)後はXP/おやつバーを「MAX」表示に切り替え

## 6. アルバム(S-30 / S-31)

### S-30 アルバム一覧
- 写真グリッド(月別セクション)。タップで詳細(写真・場所・タグ・メモ)
- 空状態: 「過去の思い出を登録しよう」+登録ボタン
- 右下FAB「+」→ S-31

### S-31 アルバム登録
1. 写真選択(ライブラリから。ここはライブラリ可 — 過去写真の登録が目的のため)
2. 場所入力(地図ピン指定 or スポット名検索)
3. 日付・タグ・メモ(任意)→ 登録
- 登録時に「アルバムは育成対象外です」を初回のみ説明(ゲームバランス保護の理由をポジティブに: 「キャラとの出会いはその場のチェックインだけ!」)

## 7. マイページ(S-40〜S-43)

### S-40 マイページ
- プロフィール(表示名・相棒・チェックイン数・図鑑達成率)
- メニュー: バッジ一覧(S-42) / ホームエリア変更(S-41) / 設定(S-43)

### S-41 ホームエリア変更
- 現在の設定地点を地図表示
- クールダウン中: 「次回変更可能日: YYYY-MM-DD」を表示しボタン無効化
- 変更可能時: 確認ダイアログ「基準地を変更すると遠出ボーナスの計算が変わります。次の変更は90日後です」

### S-43 設定
- 通知設定(週1のお知らせ ON/OFF)
- 利用規約 / プライバシーポリシー(WebView or 静的画面)
- アカウント削除: 2段階確認(「すべての記録・写真・キャラが削除されます」→ テキスト入力確認)→ Edge Function呼び出し(DATABASE.md 5.10)

## 8. 共通状態・エラーマトリクス

| 状態 | 表示 | 該当画面 |
|---|---|---|
| オフライン | 上部バナー「オフライン(同期待ちN件)」 | S-10ほか全タブ |
| 同期失敗 | バナーを赤に+タップで再試行 | S-10 |
| 位置情報権限なし | FAB無効+モーダル「設定を開く」 | S-10 |
| カメラ権限なし | 同上(メッセージはカメラ用) | S-10 |
| 1日10件到達 | FAB無効+トースト | S-10 |
| POI候補0件/API障害 | 手動入力を主導線に切替 | S-12 |
| 図鑑未入手 | シルエット+★表示 | S-20 |
| アルバム0件 | 空状態イラスト+登録導線 | S-30 |
| 初回チェックイン前 | 地図に吹き出し「最初のチェックインをしてみよう」 | S-10 |
| 抽選pending(手動入力分) | 復帰後にPOI自動解決を試行(REQUIREMENT 4章)。自動解決できない場合のみスポット詳細に「キャラ抽選待ち」バッジ+タップでPOI候補選択シート | S-11 |
