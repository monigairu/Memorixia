## 1. 世界観・デザイン原則

1. **モチーフ**: 「系統モチーフ×生き物」。どの系統で出会ったかが一目で伝わる見た目にする
2. **トンマナ**: ゆるかわ2頭身・手描き風。線は柔らかく、彩度は中〜低で写真(記録)と喧嘩しない
3. **命名**: 系統語×生き物語のカタカナ合成、3〜6文字(例: モミジカ=紅葉×シカ)
4. **進化**: ベースの姿は維持し、成長+系統アイテムが豊かになる(シルエットの連続性を保ち「同じ子が育った」と分かること)
5. **視認性**: 地図マーカー(相棒)として約48〜64pxで表示されるため、シルエットと1つの特徴色で判別できること
6. **表現形式**: **2Dイラストで確定**(3Dモデルは採用しない)。理由: トンマナ「手描き風」は2D表現であること、52種×3段階の3D制作は個人開発のコストに見合わないこと、react-native-mapsのマーカーは画像ベースで3D表示の仕組みがないこと。動きが必要になった場合も2Dスプライトの簡易アニメーションで対応する

### レア度の「格」の設計

| レア度 | 格 | デザイン指針 |
|---|---|---|
| ★1 | 身近な生き物 | 街で見かける動物。親しみ重視、色数少なめ |
| ★2 | ちょっと特別 | 小道具・衣装が付き、個性が立つ |
| ★3 | その土地の主(ぬし)感 | 風格・自然現象をまとう。色数・装飾が増える |
| ★4 | 伝説・幻想種 | 幻想生物や神格。発光・浮遊などの特別表現を1点だけ許可 |

## 3. 初期相棒の仕様(未定義だったギャップの解消)

サインアップ直後はキャラを1体も持っておらず、「現在地マーカー=相棒」が成立しない。以下のとおり定める:

- **スターター配布はしない**。最初のキャラとの出会いは初チェックインで起きる(「記録から始まる」コンセプトを守る)
- 初チェックインまでの現在地マーカーは通常の青ドット+吹き出し「最初のチェックインをしてみよう」(SCREEN.md S-10の空状態)
- **最初に入手したキャラは自動で相棒に設定**される(以降の変更はキャラ詳細から)

## 4. AI生成ワークフロー

1. **スタイルアンカーの確定**: ラテチュン(★1・1段階目)を基準1枚として作り込み、トンマナを確定する(REQUIREMENT 6章「基準1枚を確定→AI生成で展開」)
2. **プロンプトテンプレート**(全キャラ共通の骨格):
   ```
   ゆるかわいい2頭身のキャラクター、手描き風の柔らかい線、
   [生き物]がモチーフ、[系統アイテム]を身につけている、
   白背景、正面向き、全身、フラットな塗り、彩度控えめ
   + スタイルアンカー画像を参照指定
   ```
3. **生成単位**: 1キャラ=3段階を同一セッションで生成し、シルエットの連続性を担保する
4. **後処理**: 背景透過(PNG)、正方形トリミング
5. **QAチェックリスト**: □2頭身が保たれている □系統アイテムが小サイズでも視認できる □3段階が同一キャラに見える □特徴色が1色に絞れている

## 5. アセット仕様(DATABASE.md 6章と対応)

| 項目 | 内容 |
|---|---|
| ファイル命名 | `assets/characters/{slug}_s{1..3}.png`(例: `latte_chun_s1.png`) |
| slug | characters_masterのシード時に確定(例: ラテチュン → `latte_chun`) |
| サイズ | 原寸1024×1024(図鑑・詳細・演出用)。地図マーカーはアプリ側で縮小表示 |
| 形式 | PNG(背景透過) |
| 総数 | 52種×3段階=156点+シルエット版(図鑑未入手用はアプリ側でシルエット加工し、専用画像は作らない) |

## 6. 制作優先順位(仮絵→本絵の差し替え計画)

REQUIREMENT 7章「AI生成+仮絵で開発を止めない」に基づき、**ローンチ時点で156点すべて存在している必要がある**(空マスなし原則)が、品質は段階的に上げる:

| 優先度 | 対象 | 理由 |
|---|---|---|
| P0 | ★1全13種(39点)+スタイルアンカー | 出現率70%で最も目に触れる。相棒になる確率も最大 |
| P1 | ★2全13種(39点) | 出現率22% |
| P2 | ★3・★4(78点) | 出現率8%。仮絵のままでも遭遇機会が少なく、後から差し替えやすい |

仮絵: プロンプトテンプレートによる一括生成そのまま(QA省略)。本絵: QAチェックリスト通過+必要に応じ再生成。
